HyperFile Overview

In a nutshell, HyperFile is a scale-out NAS platform that provides filesystem protocols for clients and transparent data tiering to object storage (Cloudian HyperStore). Data is written by applications / clients on HyperFile and then it takes care about managing the underlying tiers of storage, leveraging its ILM capabilities.

IMAGE-1

It can be deployed as a stand-alone node or as an HA pair and provides several capabilities:

  • Data caching (multiple levels of disk) and tiering to HyperStore
  • ILM engine and Data Tiering / Migration
  • WORM (including other features like Audit, etc… for Compliance)
  • Bi-modal access to data (directly read via S3 in object storage)
  • Integrated Data Protection via Snapshots
  • Multi-site deployments and Geo-Distribution
  • AD/LDAP integration and User Quota

Thus, we do need to provide the following resources to enable successful deployment:

  • Computing resources to run the HyperFile instance (i.e. the VM)
  • One or more disk devices to be used as primary and secondary disk levels
  • Access to HyperStore via Http / S3
  • The proper HyperFile product licenses (based on features and data capacity)
  • (optional) In case of HA implementations a shared disk resource to be used as common data cache by the two HA nodes (Active / Passive implementation)
