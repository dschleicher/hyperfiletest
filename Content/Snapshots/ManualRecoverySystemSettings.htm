Manual Recovery of the System Settings

In case of a complete node failure, or a failure of the root disk device, the system configuration can be recovered from a backup.

The system backup files are generated when taking a snapshot. They look like this: 

<hostname>_sys-devices_<date>.tar.gz
<hostname>_sys-system_<date>.tar.gz
<hostname>_sys-volumes_<date>.tar.gz

For example:

alpha40_sys-devices_2018-01-18_16-54-56.tar.gz
alpha40_sys-system_2018-01-18_16-54-56.tar.gz
alpha40_sys-volumes_2018-01-18_16-54-56.tar.gz

Unzip those files, stop all services (with infistop) and then move the extracted files in the /ovmh/devices, /ovmh/system, and /ovmh/volumes directories respectively. Then restart services (with infistart).

Keep in mind that some of the other system settings (e.g. network, NTP, AD configuration, locally defined users) will not be recovered. This might include some of the networking part of the cluster configuration.
