Restoring a Backup as a Snapshot

In our example we took a few snapshots of volume “Test2_snapshot” and “Test1_snapshot”. Now, we did destroy the volume “Test1_snapshot” to show the recovery procedure (see later in this document). Let’s focus on “Test2_snapshot” and do a file recovery. The status of the current filesystem is:

Let’s delete a few files (the ones highlighted). If we want to get them back, we can mount one of the snapshots that contain them (or just guess based on the timestamp) and browse the volume.

The chosen snapshot is ready to be opened and browsed to look for data. The page contains operations that can be performed and some statistics about it. To be able to access the snapshot it's required that you open it, just like a volume. Click the Open button to do so.

Now we can mount the snapshot as a totally independent volume on any client. To manage how the snapshot is exported click the SHARING tab. The default settings will match the original volume's (except the exported name will be different). You can change these to your preference and then click Apply settings to finalize the changes.

A dialog will appear confirming that the snapshot was opened. Click Ok and the page will reload.

The snapshot can now be mounted on any client (CIFS in the example) using its own mount point: _SNAPSHOT_Test2_snapshot_20180221_054821. In the example:

\\192.168.153.100\_SNAPSHOT_Test2_snapshot_20180221_054821

And we got back all the files that were deleted previously.

In this case, being mounted on Windows, we can also notice that the file icon is slightly different. The reason is that the files are not on disk. In fact, snapshot just allowed restore the pointers and metadata without repopulating the whole disk cache.

This is a very useful way to avoid space waste and long file transfers. We just access what we really need (1 file, 1 folder, etc…). When we click on a file, we can see it physically restored to disk.

When you are done, close the snapshot by clicking the Close button next to "Snapshot status" in the overview page and then click Delete snapshot to delete the snapshot. Delete doesn’t delete the snapshot, just removes it from the list of mounted snapshots (the rules about number of versions keep snapshot in the system). We can remount it when needed.
