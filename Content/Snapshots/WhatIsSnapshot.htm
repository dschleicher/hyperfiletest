What Is a Snapshot?

Snapshot is defined as a point-in-time image of the content of a data repository.

By using this approach HyperFile can provide system administrators a powerful tool to protect data by defining a schedule of planned backups or run them on the spot in case of need. In the specific case of HyperFile and HyperStore, a snapshot allows to freeze the pointers to data in HyperStore and leverages versioning to enable multiple point-in-time backup to be taken.

Thus, in current form, snapshot works only with HyperStore (or other object storage) with versioning enabled on buckets used by HyperFile. In other terms, when a snapshot is taken, HyperFile pointers to specific HyperStore object versions are saved.

Snapshots are sometimes referred as "volume backups" since they can also be used to recover the volume in case the database is lost or corrupted. This includes the volume configuration and the file metadata, but it does not include copies of the data.

It’s important to note that snapshots drive higher space consumption in the object storage, so we need to account for this in design phase. See the “Storage efficiency considerations” chapter of this document.
