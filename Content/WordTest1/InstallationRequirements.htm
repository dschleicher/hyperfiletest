HyperFile Installation Requirements overview

The VM is provided by Cloudian in form of an OVA file. The underlying hardware resources must be made available for installation. Installation pre-requirements are highlighted in this document Appendix A. 

Getting the HyperFile software

The main steps to follow are:

  • Download HyperFile OVA:

For existing customer and partners or Cloudian personnel follow the instructions in this KB article (needs a Salesforce account):

00000414 Cloudian HyperFile Product Package/Documentation Download

For everybody else: please contact a Cloudian representative to get access to the software.

  • HyperFile license: HyperFile comes with a pre-configured license that includes 512 GB disk and 512 GB Object Storage tier and unlimited time. For other features a license is needed and can be either generated by Cloudian personnel for PoC / testing or acquired as a part of your software subscription (contact your sales representative for more details).

Provisioning Resources

  • Provide ESXi platform: can be a dedicated or share platform. HyperFile is scalable so the amount of resources can really be adjusted to real needs. Appendix B provides some rough sizing information as part of the site preparation checklist.

 

  • Provide disk storage for Data cache and Database. In short there are two types of devices we can present to HyperFile:

 

  • VMDK (that can be anything, but it’s managed by hypervisor so transparent to HyperFile)
  • NAS share

 

Note that the OVA come with some devices configured by default.

These devices are pretty much the bare minimum to ensure HyperFile can work and mostly designed for PoC / Test. For production use, need to be expanded (or new devices added) as described later in this guide. 

HyperFile allows a lot of flexibility in device configuration so it’s possible to add/modify/resize / depopulate / etc... at almost any time, even in case of mistakes in initial sizing or different workloads.

 

A rough sizing for disks is highlighted in Appendix A.

Cloudian provides a separate sizer for fine-tuning the configuration.

 

  • Prepare Networking. It’s critical that the proper networks are in place.

 

Because of the use of hypervisor, there is a lot of flexibility in using VLANs and link aggregation that are completely transparent to HyperFile. Network sizing is out of the scope of this document, but we need to keep in mind following things:

 

  • HyperFile needs to communicate with the Clients /Applications for data I/O (i.e. “Front-end”)
  • HyperFile needs to communicate with the Object Storage (HyperStore in this case) for data I/O (i.e. “Back-end”)

 

In small deployments (like PoCs or tests) Front-end and Back-end can be on same network.

 

  • (optional) HyperFile needs to communicate with the disks (cache) for data I/O (i.e. “Back-end”) if we use NAS devices as cache.
  • (optional) HyperFile needs to communicate with its peer in case of HA deployment. In this case the networking gets a bit more complicated (see Appendix A)

 

  • (optional) Provide HyperStore access.

 

HyperFile installation, per se, works even without Object Storage. In this case only disk cache will be used. However, for the completeness of the installation, we do assume that HyperStore is correctly installed and configured and that the following information are available.

 

  • HyperStore access URL
  • One of more buckets to be used by HyperFile
  • S3 Access credentials (access key / secret key)
